Creating Tools and Experimenting

Posted: 11-26-2025

Updated: 11-29-2025

Recently I've picked up a small habit of doing 4 github commits per day, especially using those four commits in slowly progressing my small personal projects, and as well as progressing in learning more languages through tutorials and sample code. Back then, I felt bad with feeling the need to commit per day, because it felt like I've done some petty git commits for the sake of keeping up records. I find it strange that this time, I do not feel that, and that this case, while similar to the past, is completely different.

Hm... how should I put it, each commit I do (in those 4 commits) makes sure the projects / repos I'm committing to *stays* in my head, even if in the background. I actually find myself wanting to tackle those small projects and building my personal learning repo. I find it fascinating that simply building momentum creates this sort of intrinsic value or satisfaction in making small software or medium-scaled software in very small step by step bits.

Because of this I actually want to make a list of project ideas that I want to tackle. so that i can tackle that list in a step by step manner just like as I tackle a single project in a step by step manner.

In the end, it is all momentum, a habit. And thus, Atomic Habits by James Clear once again rings in my head.
